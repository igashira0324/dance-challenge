判定精度向上のためのガイド

ダンエボのような「体感」を実現するための新しい判定ロジック

1. なぜ「関節角度」なのか？

これまでのオイラー角比較は、カメラの角度や向きに弱く、判定が不安定でした。
新しいロジックでは、MediaPipeのランドマークからベクトル演算を行い、関節の曲がり具合（内積）を直接計算します。

graph TD A[Landmark A: 肩] --> V1[ベクトル BA] B[Landmark B: 肘] --> V1 B --> V2[ベクトル BC] C[Landmark C: 手首] --> V2 V1 & V2 --> D[内積計算] D --> E[関節角度 ラジアン] E --> F[ターゲット角度と比較] F --> G{判定 PERFECT/GOOD/MISS}

2. 新しい判定パラメータ

以下の関節を主要な判定対象としています：

  • Elbow: 肘の曲がり具合
  • Shoulder Lift: 肩の上がり具合（脇の開き）
  • Arm Open: 腕の前後・左右への広がり

しきい値は現在 0.35 rad (約20度) 以内をPERFECTとしています。

3. 【重要】ポーズのキャプチャ機能

開発中に「理想のポーズ」を定義するためのショートカットを追加しました。

Alt + C

1. アプリを起動してカメラをオンにします。
2. 理想的なポーズをとった状態で Alt + C を押します。
3. ブラウザのコンソールに JSON 形式で角度データが出力されます。
4. その値を constants/index.ts の P 定義に貼り付けるだけで調整完了です！
