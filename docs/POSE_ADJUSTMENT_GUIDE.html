判定精度向上のためのガイド

ダンエボのような「体感」を実現するための新しい判定ロジック

1. 実測値のキャプチャ (Alt + C)

理論値ではなく、実際のあなたのカメラ環境での数値を採用するのが最も確実です。

  1. ゲームを起動し、カメラをオンにします。
  2. 判定したいポーズ（例：万歳）をカメラの前で取ります。
  3. そのまま Alt + C キーを押します。
  4. ブラウザのコンソール（F12）を開くと、以下のようなJSONが出力されます：
{
  "leftArmElbow": 3.12,
  "rightArmElbow": 3.08,
  "leftShoulderLift": 2.85,
  "rightShoulderLift": 2.91
}

これを src/constants/index.ts の P.BANZAI などにコピー＆ペーストしてください。

2. リアルタイム・デバッグ

開発モード（npm run dev）では、マーカー出現中に現在の姿勢との「差異（Diff）」がコンソールにリアルタイム表示されます。

  • PERFECT基準: Diff < 0.35 (約20度以内の誤差)
  • GOOD基準: Diff < 0.70 (約40度以内の誤差)

「完璧にポーズを取っているのにMISSになる」場合は、コンソールのDiff値を確認し、しきい値を調整するか、ターゲット角度を実測値に更新してください。

3. シルエット・ガイド

画面に現れる緑色のシルエットは、targetPoseAngles に定義された角度をリアルタイムにVRMモデルへ復元して表示しています。

もしシルエットの腕の向きがおかしい場合は、ボーンの回転軸マッピング（vrmService.ts の applyPoseFromPoseAngles）を調整する必要があります。

4. Ripple（座標）マーカーの注意点

カメラ映像は「ミラー（左右反転）」表示されています。本システムでは自動的にミラー補正を行っているため：

  • 画面の右側に表示されるマーカーは、あなたの右手を出すことで反応します。
  • x: 0.85 は画面右端、x: 0.15 は画面左端を指します。
